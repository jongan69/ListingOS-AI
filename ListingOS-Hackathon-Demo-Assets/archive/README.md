# Archive

Superseded material, retained for audit and reasoning history. **Nothing here is a
current plan.** Do not produce against these documents.

Current production documents are [`../PRODUCTION_PLAN.md`](../PRODUCTION_PLAN.md) and
[`../DEMO_VIDEO_SCRIPT_V2.md`](../DEMO_VIDEO_SCRIPT_V2.md).

## `planning/` — pre-production, superseded

Written before the script and the two-track model existed. They describe deliverables
that were never made.

| File | What it was | Why it is here |
| --- | --- | --- |
| `edit-plan.md` | Specs for 30s / 60s / 90s / vertical cuts | Only one cut is being made, at ~2:53. The multi-length plan was abandoned. |
| `demo-shot-list.md` | Early shot list and capture order | Superseded by `PRODUCTION_PLAN.md` §3 and §6. |
| `shot-plan-final.md` | Nine-beat table mapping shot → Rotato treatment → caption | Superseded by the beat map in `PRODUCTION_PLAN.md` §3, which uses the same shot-to-treatment structure against the current script. |

**What survived:** the truth guardrails and recording rules from these files were
consolidated into `PRODUCTION_PLAN.md` §7. In particular, "do not imply a fresh live
publish unless the recording proves it" originated here and is now the governing rule
for the open publish blocker (`PRODUCTION_PLAN.md` §4).

## `automated-pass/` — the fully automated render attempt

| File | What it was |
| --- | --- |
| `FINAL_VIDEO_NOTES.md` | Production record for the Remotion and Rotato masters |

These renders were produced while the product was incomplete and the footage required
for a complete functional demo did not exist. Polish was standing in for proof. The
approach was abandoned in favor of the two-track model, where live-action evidence is
the spine and automated material is overlay.

The *tooling* from this pass remains current and is documented in its live locations:
[`../remotion/`](../remotion/), [`../clipcaption-projects/`](../clipcaption-projects/),
and [`../rotato-exports/`](../rotato-exports/). Only the conclusion — that the automated
render was the master — is obsolete.

Technical measurements in `FINAL_VIDEO_NOTES.md` describe files that no longer exist and
should not be cited as current specifications.
